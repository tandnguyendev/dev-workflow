# Project map

*What EXISTS in this project and WHERE — the map agents read before proposing to
build something. `conventions.md` says how to write code here; this file says what
is already written. Keep the two apart: a rule goes there, a location goes here.*

> Generated by `/dev-workflow:init` and kept current by `/dev-workflow:feature`,
> which appends each shipped feature at Stage 5. Re-run `init` to re-derive it
> after a big refactor.
>
> Unlike `conventions.md` — which every subagent re-reads in full, every phase —
> this file is read LAZILY: the orchestrator reads it when researching and
> planning, the coder reads the entries for the modules it is touching. So it may
> be longer, but it still earns its length: entries are pointers, not
> documentation. Say where a thing lives and what it is for; do not restate what
> the code says. Aim for one to three lines per entry, and delete an entry the
> moment it stops being true — a stale map is worse than no map, because agents
> trust it.
>
> Delete the guidance blockquotes once filled.

## Shared building blocks
> The things a new feature is expected to REUSE — base classes, middleware,
> validators, error types, the HTTP/DB client, auth helpers, test fixtures,
> config access. Name the file and when to use it. This section is what stops
> agents writing a fourth retry helper.

## Extension points
> The seams to hook into when adding something: how a route/command/handler/job
> gets registered, where a migration goes, how a new config key is declared, how a
> feature flag is added. "To add X, touch Y" — one line each.

## Known gotchas
> Traps that are not visible from the code being edited: a module that must be
> imported in a certain order, a cache that must be invalidated with a write, a
> table two services both write to, a test that only passes with a fixture loaded.
> Each line should have burned someone at least once.

## Glossary
> Domain term -> what it is called in the code, where they differ (e.g. "top-up"
> -> `WalletCredit`). Skip if the names match.
